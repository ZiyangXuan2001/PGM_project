Current Project Summary For LaTeX Report
=======================================

Date written: 2026-05-28
Project: PGM_project
Task: Diving48 V2 video classification using frozen visual features and lightweight temporal / PGM heads.


1. Executive Summary
--------------------

This project studies whether probabilistic graphical model (PGM) smoothing can
improve video classification when the visual backbone is frozen. The target
dataset is Diving48 V2, a fine-grained diving action dataset with 48 classes.
The main input representation is frozen CLIP ViT-B/16 frame embeddings:

    X: [B, T, D] = [B, 16, 512]

where each video is represented by 16 uniformly sampled frame features.

The central PGM idea is to treat a temporal sequence of feature vectors as noisy
observations and infer a smoother latent trajectory using a Gaussian chain PGM.
The Gaussian chain has the MAP objective

    Z* = argmin_Z [
        alpha * sum_t ||Z_t - X_t||^2
        + lambda * sum_t ||Z_t - Z_{t-1}||^2
    ].

Equivalently, for a temporal path graph Laplacian L,

    (alpha I + lambda L) Z = alpha X.

Several model families were implemented and tested:

1. E-series controlled ablations:
   - DiffNet + optional GaussianPGMSmoother + optional InformationAccumulator.
   - Best single-seed PGM-related result:
       feature-side learnable PGM + E3 post-DiffNet PGM evidence
       best validation top-1 = 22.23%.

2. P-series simplified trajectory-matrix ablations:
   - A cleaner report-friendly family with no attention classifier and no
     InformationAccumulator.
   - Tests no PGM vs pre-trajectory PGM vs post-trajectory PGM.
   - Multi-seed result supports the main PGM claim:
       noPGM mean best val top-1 = 15.27%
       prePGM lambda=0.30 mean best val top-1 = 16.16%
       postPGM lambda=0.80 mean best val top-1 = 16.51%

Recommended report focus:

    Use the P-series as the main clean PGM experiment because it is simpler,
    controlled, and multi-seed. Use the E-series as an extended model family
    showing that stronger neural heads can reach higher validation accuracy,
    but the PGM contribution there is harder to isolate.

3. Core Gaussian PGM
--------------------

The Gaussian temporal smoother is a fixed MAP inference layer over a path graph.
For a sequence S = [S_1, ..., S_T], it infers latent states Z by minimizing

    E(Z) = alpha * sum_t ||Z_t - S_t||^2
           + lambda * sum_t ||Z_t - Z_{t-1}||^2.

Here:

    alpha controls observation fidelity.
    lambda controls temporal smoothness.
    L is the temporal path graph Laplacian.
    A = alpha I + lambda L is the Gaussian information matrix.

The closed-form MAP solve is

    Z* = A^{-1} alpha S.

Implementation notes:

    - Fixed Gaussian smoothing has zero trainable parameters.
    - Some E-series runs also support learnable lambda, which adds only a scalar.
    - The PGM layer preserves tensor shape.
    - The PGM layer is differentiable, but in fixed-lambda P-series experiments
      it does not add model capacity.


4. E-Series Model Family
------------------------

The E-series is the original controlled DiffNet + PGM + information accumulator
family. It is useful because it directly tests the proposed PGM evidence idea,
but some later E-series variants use a large temporal attention classifier, so
PGM contribution must be interpreted carefully.

Common CLIP input:

    X: [B, 16, 512]

Most CLIP E-series runs use:

    d_y = 128
    DiffNet hidden_dim = 256
    InformationAccumulator K = 8
    InformationAccumulator d_h = 128
    num_classes = 48


4.1 PairwiseDiffNet
-------------------

For adjacent frames:

    x_t:      [B, 15, 512]
    x_next:   [B, 15, 512]
    diff:     x_next - x_t
    abs_diff: |x_next - x_t|

The original PairwiseDiffNet input is

    [x_t, x_next, diff, abs_diff] -> [B, 15, 2048].

Default stack:

    Linear(2048 -> 256)
    GELU
    LayerNorm(256)
    Dropout
    Linear(256 -> 128)

Output:

    R: [B, 15, 128]

Interpretation:

    R_t is a learned pairwise temporal observation/relation feature between
    frames t and t+1.


4.2 E-Series Ablation Definitions
---------------------------------

E0: feature_mean

    X -> mean over time -> classifier -> logits

Purpose:

    Frozen feature baseline. No temporal relation modeling, no DiffNet, no PGM,
    no accumulator.

E1: diff_mean

    X -> PairwiseDiffNet -> R -> mean_t(R_t) -> classifier -> logits

Purpose:

    Tests whether learned adjacent-frame temporal differences help over mean
    CLIP features.

E1.5: diff_info_accum

    X -> PairwiseDiffNet -> R -> InformationAccumulator -> H_final
      -> classifier -> logits

Purpose:

    No-PGM accumulator control. This is essential because E3 uses an
    InformationAccumulator, so E1.5 separates the effect of the accumulator from
    the effect of PGM inference.

E2: diff_pgm_mean

    X -> PairwiseDiffNet -> R -> GaussianPGMSmoother -> Y
      -> mean_t(Y_t) -> classifier -> logits

Purpose:

    Tests whether naive Gaussian smoothing of DiffNet outputs helps when the
    classifier only sees the mean smoothed relation.

Important observation:

    E2 often matches E1 when classification uses only mean pooling, because the
    Gaussian smoother tends to preserve or weakly change the temporal mean.

E3: diff_pgm_info_accum

    X -> PairwiseDiffNet -> R -> GaussianPGMSmoother -> Y
      -> local PGM evidence U -> InformationAccumulator -> H_final
      -> classifier -> logits

Purpose:

    Main PGM evidence model. It tests whether PGM MAP inference provides useful
    local evidence beyond the raw DiffNet observations.


4.3 E3 Local PGM Evidence
-------------------------

For each time step t:

    delta_t = Y_t - R_t
    abs_delta_t = |Y_t - R_t|

    residual_t =
        mean over feature dim of (Y_t - R_t)^2
        shape [B, 1]

    smoothness_t =
        mean over feature dim of (Y_t - Y_{t-1})^2
        shape [B, 1]

For t = 0, smoothness_t is set to zero because there is no previous Y.

The E3 evidence vector is

    u_t = concat[R_t, Y_t, abs_delta_t, residual_t, smoothness_t].

If d_y = 128:

    dim(u_t) = 3 * 128 + 2 = 386
    U: [B, 15, 386]

If d_y = 64:

    dim(u_t) = 3 * 64 + 2 = 194.


4.4 InformationAccumulator
--------------------------

The InformationAccumulator maintains a learned information state

    H: [B, K, d_h] = [B, 8, 128]

and updates it sequentially using either R_t in E1.5 or U_t in E3.

Important conceptual clarification:

    The true Gaussian PGM information matrix is A = alpha I + lambda L.
    The learned InformationAccumulator is not the same as A. It is a learned
    sequential evidence accumulator. E1.5 controls for this accumulator, and E3
    tests whether PGM-inferred local evidence improves the accumulator input.


4.5 Feature-Side Frame PGM
--------------------------

A later idea was that smoothing after DiffNet may be conceptually questionable:
DiffNet outputs directional pairwise relation vectors, and those vectors may not
need to be smooth. For example, real diving motion can change direction or phase
quickly. Therefore a more natural PGM placement is before DiffNet, directly on
the frozen frame observations.

Feature-side PGM formulation:

    X_t: frozen CLIP frame observation
    Z_t: latent cleaned frame state

    Z* = argmin_Z [
        alpha * sum_t ||Z_t - X_t||^2
        + lambda * sum_t ||Z_t - Z_{t-1}||^2
    ]

Then:

    X: [B, 16, 512]
    -> FrameGaussianPGMSmoother
    -> Z: [B, 16, 512]
    -> PairwiseDiffNet(Z_t, Z_{t+1})
    -> R: [B, 15, 128]

Feature-side PGM can be combined with E1, E1.5, or E3:

    feature PGM + E1:
        Z -> DiffNet -> mean(R) -> classifier

    feature PGM + E1.5:
        Z -> DiffNet -> accumulator(R) -> classifier

    feature PGM + E3:
        Z -> DiffNet -> R -> post-DiffNet PGM -> U -> accumulator(U)

The strongest E-series PGM result uses both:

    feature-side learnable PGM before DiffNet
    plus post-DiffNet PGM evidence inside E3.

7. P-Series Simplified Model Family
-----------------------------------

The P-series was created to produce a cleaner PGM-centered experiment. It
removes the large temporal evidence attention classifier and removes the
InformationAccumulator. This makes the comparison more controlled:

    same trainable neural architecture
    same parameter count
    same optimizer and data
    only PGM placement/lambda changes

Common input:

    X: [B, 16, 512]

Architecture components:

1. TemporalProjection

    Per-frame projection with no temporal mixing:

        LayerNorm(512)
        Linear(512 -> 128)
        GELU
        LayerNorm(128)

    Output:

        U: [B, 16, 128]

2. Optional pre-trajectory PGM

    Applied before TemporalProjection:

        X -> GaussianFramePGMSmoother -> X_smooth -> TemporalProjection

    In the implementation, prePGM smooths the raw CLIP sequence X.

3. Optional post-projection / pre-DiffNet PGM

    Applied after TemporalProjection:

        X -> TemporalProjection -> U -> GaussianFramePGMSmoother -> Z

    This is called postPGM in the P-series notes because it happens after the
    frame projection and before the trajectory DiffNet.

4. ProjectedPairwiseDiffNet

    Adjacent pair construction:

        u_t:        [B, 15, 128]
        u_next:     [B, 15, 128]
        diff:       u_next - u_t
        abs_diff:   |u_next - u_t|

        concat [u_t, u_next, diff, abs_diff] -> [B, 15, 512]

    Network:

        Linear(512 -> 256)
        GELU
        LayerNorm(256)
        Dropout(0.25)
        Linear(256 -> 128)

    Output:

        R: [B, 15, 128]

5. TrajectoryMatrixClassifier

    Flatten temporal relation matrix:

        R: [B, 15, 128]
        flatten(R): [B, 1920]

    Classifier:

        LayerNorm(1920)
        Dropout(0.3)
        Linear(1920 -> 48)

    Output:

        logits: [B, 48]

Trainable parameter count:

    All trajectory_matrix_linear P-series variants:
        327,728 trainable parameters.

    Classifier parameters:
        96,048 trainable parameters.

    PGM parameters:
        0 trainable parameters for fixed-lambda PGM.

This is a clean comparison because adding the PGM does not add trainable
capacity.

9. Main Scientific Interpretation
---------------------------------

The strongest report-friendly conclusion is:

    A lightweight Gaussian-chain PGM can improve frozen-feature video
    classification when placed at a suitable temporal representation level.

More detailed interpretation:

1. Pairwise temporal modeling helps.

    CLIP E1 diff_mean improved strongly over E0 feature_mean:

        E0: 11.93%
        E1: 17.37%

    This shows that adjacent-frame temporal relation features are useful for
    Diving48.

2. Naive post-DiffNet smoothing plus mean pooling is weak.

    E2 did not clearly improve over E1. This suggests that simply smoothing R
    and taking mean(Y) hides the PGM signal.

3. PGM evidence is more useful when exposed to a stronger temporal classifier.

    E3 with temporal evidence attention slightly improved over E1.5 attention:

        E1.5 attention: 21.76%
        E3 attention:   22.04%

    The gain is small, but directionally supports the PGM evidence design.

4. Feature-side PGM is conceptually cleaner but not automatically better.

    Smoothing raw CLIP features before DiffNet is theoretically appealing
    because CLIP frame embeddings are direct observations, whereas DiffNet
    outputs are directional relation vectors.

    Empirically:

        fixed lambda=1 was almost neutral,
        fixed lambda=5 hurt,
        learnable frame PGM stayed weak,
        feature-side PGM + E3 reached 22.23%.

5. P-series gives the cleanest PGM evidence.

    The P-series removes the large attention classifier and keeps parameter
    counts identical. In that setting, PGM smoothing improved the mean validation
    top-1 across 3 seeds:

        noPGM:        15.27%
        prePGM:       16.16%
        postPGM:      16.51%

    Therefore, the best course-project story is not "we reached 60% accuracy."
    The better story is "we designed controlled PGM ablations and found a
    consistent PGM benefit under a clean fixed-capacity setup."


10. Why Accuracy Is Still Far Below 60%
---------------------------------------

The current best validation top-1 is far below 60%. Likely reasons:

1. Frozen global frame embeddings are limited.

    CLIP global pooled embeddings compress each frame into one vector. Diving48
    requires fine-grained pose, rotation, takeoff, twist, and entry-phase cues.
    These details may be weakly represented in global CLIP features.

2. No end-to-end video backbone is trained.

    The visual backbone is frozen. The trainable head must infer action classes
    from precomputed frame-level features only.

3. Temporal sampling is sparse.

    Each video is represented by 16 sampled frames. Some diving actions differ
    by subtle temporal phase or body configuration, which may be missed.

4. Diving48 is fine-grained.

    Many classes are visually similar, and the difference can depend on the
    number of rotations/twists or body pose during short windows.

5. Lightweight heads underfit visual complexity or overfit validation.

    The larger E-series attention heads achieved higher validation accuracy but
    showed overfitting: training accuracy rose strongly while validation loss
    increased after the best epoch.

6. PGM smoothing is not a substitute for missing visual information.

    PGM can improve temporal consistency, but it cannot recover pose details
    that are absent from the frozen representation.

12. Suggested Abstract Draft
----------------------------

This project investigates whether probabilistic graphical model smoothing can
improve fine-grained video classification when using frozen visual features.
We study Diving48 V2 using frozen CLIP ViT-B/16 frame embeddings sampled at 16
frames per video. We model the temporal feature sequence with a Gaussian-chain
PGM, where latent states are inferred by minimizing an observation-fidelity term
and a temporal smoothness term over a path graph. We evaluate both a larger
DiffNet-PGM-InformationAccumulator model family and a simplified P-series
trajectory-matrix model designed for controlled PGM ablations. In the clean
P-series setting, all variants have the same 327,728 trainable parameters and
the fixed PGM adds zero trainable parameters. Across three random seeds, the
no-PGM trajectory baseline obtains 15.27% mean best validation top-1, while
pre-trajectory PGM smoothing with lambda=0.30 obtains 16.16% and post-projection
PGM smoothing with lambda=0.80 obtains 16.51%. These results suggest that
Gaussian-chain PGM inference can provide a modest but consistent benefit for
temporal classification from frozen frame features. A larger E-series model
with feature-side learnable PGM and PGM evidence accumulation reaches 22.23%
single-seed validation top-1, but its larger attention classifier makes it less
clean as the main causal PGM comparison.


13. Suggested Method Text
-------------------------

Let X = (X_1, ..., X_T) denote the frozen frame feature sequence for a video,
where T=16 and X_t in R^512 is a CLIP ViT-B/16 frame embedding. We use a
Gaussian-chain PGM to infer temporally smoothed latent states Z by solving

    Z* = argmin_Z alpha sum_t ||Z_t - X_t||_2^2
                  + lambda sum_t ||Z_t - Z_{t-1}||_2^2.

This is equivalent to MAP inference in a Gaussian Markov random field on a
temporal path graph. If L is the path graph Laplacian, the MAP solution is
obtained by solving

    (alpha I + lambda L) Z = alpha X.

The smoother is deterministic and introduces no trainable parameters when
lambda is fixed.

For the simplified P-series experiments, each frame is first projected from
512 dimensions to 128 dimensions using a per-frame MLP. Adjacent projected
states are then converted into pairwise relation features by concatenating the
two states, their difference, and the absolute difference. The resulting
relation matrix R in R^{15 x 128} is flattened and classified with a linear
trajectory-matrix classifier. We compare three PGM placements: no PGM,
pre-trajectory PGM applied to CLIP frame observations before projection, and
post-projection PGM applied after frame projection but before pairwise relation
construction.


14. Suggested Results Text
--------------------------

The cleanest evidence for the value of PGM smoothing comes from the P-series
multi-seed experiment. Because all P-series variants have identical trainable
parameter counts and the fixed Gaussian PGM has zero trainable parameters, this
comparison isolates the effect of temporal MAP smoothing. The no-PGM baseline
achieves 15.27% mean best validation top-1 across three seeds. Applying the PGM
before trajectory extraction improves this to 16.16%, while applying the PGM
after frame projection and before pairwise trajectory construction improves it
to 16.51%. Thus, both PGM placements improve over the no-PGM baseline, with
post-projection smoothing giving the strongest result.

The larger E-series experiments show that additional temporal modeling capacity
can improve accuracy further. The best E-series single-seed run combines
feature-side learnable PGM smoothing, post-DiffNet PGM evidence, an
InformationAccumulator, and a temporal evidence attention classifier. It reaches
22.23% validation top-1. However, this model has substantially more trainable
parameters and a large attention classifier, so the P-series remains the cleaner
PGM ablation for the main report claim.


15. Suggested Discussion Text
-----------------------------

The results suggest that the location of PGM smoothing matters. Directly
smoothing final class evidence is not the goal; rather, the PGM should smooth a
temporal representation where neighboring states are expected to vary smoothly.
The P-series post-projection smoother likely works well because the projected
frame states are lower-dimensional and motion-relevant, making them easier to
regularize than raw CLIP embeddings. Pre-trajectory smoothing also helps, but
it may be less effective because CLIP frame embeddings can contain semantic
changes that should not be overly smoothed. Excessive smoothing can suppress
valid temporal transitions, which explains why lambda must be tuned.

At the same time, the absolute accuracy remains modest. This should not be
interpreted as a failure of the PGM idea alone. The experiments use frozen
global frame embeddings, while Diving48 requires subtle pose, rotation, and
phase cues. A Gaussian PGM can improve temporal consistency, but it cannot
recover visual information that is missing from the frozen representation.

17. Final Report Recommendation
-------------------------------

For the final course report, the cleanest story is:

    1. Start with frozen CLIP frame embeddings for Diving48.
    2. Introduce Gaussian-chain PGM smoothing as MAP inference on a temporal
       path graph.
    3. Present the P-series trajectory-matrix model as the controlled main
       experiment.
    4. Show that fixed PGM smoothing improves validation top-1 without adding
       trainable parameters.
    5. Use E-series as an extended model showing a higher single-seed result,
       but be honest that its PGM contribution is harder to isolate because of
       the large attention classifier.

Best concise claim:

    In a fixed-capacity trajectory-matrix model using frozen CLIP ViT-B/16
    frame features, Gaussian-chain PGM smoothing improves mean best validation
    top-1 from 15.27% without PGM to 16.51% with post-projection PGM smoothing
    across three seeds. This provides controlled evidence that PGM temporal MAP
    inference can improve frozen-feature video classification.
